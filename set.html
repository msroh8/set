الدرس الأول: القوة الخارقة لحاوية الـ Set

التأسيس والتهيئة للمسابقات البرمجية (الأولمبياد المعلوماتي).

🎮 التمهيد الجاذب (تشبيه الصندوق السحري)

"تخيلوا أننا ننظم بطولة ألعاب إلكترونية في المدرسة، وفتحنا باب التسجيل. بعض الطلاب تحمسوا وضغطوا على زر التسجيل أكثر من مرة! الآن لدينا قائمة طويلة جداً فيها أسماء وأرقام مكررة وبشكل عشوائي. في المسابقات البرمجية، إذا حاولنا تنظيف هذه القائمة وترتيبها باستخدام المصفوفات العادية (Arrays)، سنكتب سطوراً كثيرة وسنضيع وقتاً ثميناً. هنا يتدخل بطلنا اليوم: الـ set."

الـ set ليس مجرد مصفوفة، بل هو "صندوق سحري ذكي" يتميز بثلاث قوى خارقة:

  • قوة الفلترة (لا للتكرار): لديه حارس صارم يمنع دخول أي نسخة مقلدة.
  • قوة الترتيب التلقائي: أي شيء ترميه داخل هذا الصندوق، سيقوم بترتيبه فوراً (من الأصغر للأكبر).
  • قوة السرعة: يبحث عن العناصر بسرعة البرق باستخدام خوارزميات تعمل بوقت O(log n).

تفكيك الكود (خطوة بخطوة)

1. استدعاء الأداة (المكتبة الخاصة بها):

  • #include <iostream>: مكتبة لطباعة المخرجات على الشاشة.
  • #include <set>: أمر استدعاء مكتبة الحاوية الذكية (set) لنتمكن من استخدامها.
  • using namespace std;: لتسهيل كتابة الأوامر دون الحاجة لتكرار كلمة std:: قبل كل أمر طباعة.
#include <iostream>
#include <set> // هذا هو أمر الاستدعاء للأداة من المكتبة
using namespace std;

2. إنشاء الصندوق وتعبئته بأرقام عشوائية ومكررة:

  • set<int>: يخبر الكمبيوتر أننا نريد إنشاء حاوية ذكية مخصصة للأرقام الصحيحة فقط.
  • numbers: هو الاسم الذي اخترناه لهذه الحاوية.
  • {5, 2, 9, 2, 1}: تزويد الحاوية بقيم أولية. سيقوم الـ set تلقائياً بحذف الرقم (2) المكرر وترتيب البقية.
int main() {
    // أنشأنا set ووضعنا فيه 5 أرقام، لاحظوا أن الرقم 2 مكرر
    set<int> numbers = {5, 2, 9, 2, 1};

3. استعراض المحتويات وطباعتها:

  • for (int x : numbers): حلقة تكرارية حديثة تقرأ كالتالي: "لكل عنصر سمّه x متواجد داخل الصندوق numbers".
  • cout << x << " ";: أمر طباعة يطبع العنصر الحالي x وبجانبه مسافة فارغة.
    // اقرأها هكذا: "بالنسبة لكل رقم x موجود داخل الصندوق numbers..."
    for (int x : numbers) {
        cout << x << " "; // "...قم بطباعته مع مسافة"
    }
    return 0;
}

شاشة المخرجات (Output):

1 2 5 9

تحليل: اختفت الـ 2 المكررة بفضل (الحارس الصارم)، وظهرت الأرقام مرتبة بفضل (الترتيب التلقائي).

💡 نصيحة مسابقات (Golden Rule)

في أي مسألة برمجية يُطلب منك فيها إيجاد "العناصر المميزة" (Distinct elements)، أو يُطلب منك البحث السريع جداً في قائمة أرقام تتغير باستمرار، يجب أن يتبادر الـ set إلى ذهنك فوراً كأول وأقوى خيار للحل!

الدرس الثاني: كيف نصنع الصندوق الذكي؟

تعلم كيفية استدعاء الحاوية، تخصيصها، وتهيئتها.

مصنع الصناديق 🏭

تخيلوا أن لدينا مصنعاً ذكياً، قبل أن نطلب منه صنع صندوق للحفظ، سيطرح علينا سؤالاً واحداً مهماً: "ما هو نوع الأشياء التي ستضعونها في هذا الصندوق؟"
في البرمجة، يجب أن نضع "ملصقاً" يحدد نوع البيانات: أرقام؟ نصوص؟ أم حروف؟

1. تصنيع الصناديق وتحديد نوعها < >

  • set<int>: تنشئ صندوقاً يقبل الأرقام الصحيحة فقط (مثل: 5, -10, 100).
  • set<string>: تنشئ صندوقاً مخصصاً للنصوص والكلمات (مثل أسماء اللاعبين).
  • الكلمات mySet و names هي مجرد أسماء متغیرات يمكنك تغييرها كما تشاء.
set<int> mySet;              // صندوق فارغ للأرقام الصحيحة
set<string> names;           // صندوق فارغ للنصوص (أسماء اللاعبين)

2. الصندوق المجهز مسبقاً { }

نستخدم الأقواس المعقوفة لتهيئة الـ set بقيم ابتدائية لحظة شرائه:

  • char: نوع بيانات مخصص للحروف المفردة.
  • {'a', 'b', 'c'}: بدلاً من إنشاء صندوق فارغ وإضافة الحروف واحداً تلو الآخر، قمنا بإنشائه وتعبئته في نفس السطر لاختصار الوقت.
set<char> letters = {'a', 'b', 'c'}; // صندوق للحروف وبداخله 3 حروف جاهزة

💡 نصيحة مسابقات (Pro Tip)

اختيار النوع الصحيح للـ set يغير طريقة الترتيب التلقائي!
إذا استخدمت set<string> ووضعت أسماء ("Ziad", "Ali", "Badr")، فإن الـ set ذكي بما يكفي ليرتبها أبجدياً: "Ali" ثم "Badr" ثم "Ziad". في المسابقات، هذا سيوفر عليك كتابة دوال الترتيب المعقدة.

الدرس الثالث: لوحة التحكم والعمليات

تعلم الأوامر الأساسية للتحكم في حاوية الـ set واستخدامها بذكاء وسرعة.

تخيلوا أنكم تجلسون في "غرفة التحكم". أمامكم لوحة تحكم تحتوي على أزرار أساسية. سنقسمها إلى ثلاث مجموعات رئيسية لتسهيل حفظها أثناء المسابقات:

المجموعة الأولى: أسلحة التعديل ⚔️

  • زر الإضافة insert(x): إدخال عنصر جديد. إذا كان موجوداً مسبقاً سيتجاهله.
    • s.insert(5): تأمر الحاوية بإضافة الرقم 5. الحاوية سترتب الرقم في مكانه الصحيح تلقائياً وترفضه إذا كان موجوداً مسبقاً.
    s.insert(5); // رمينا الرقم 5 داخل الصندوق
  • زر الممحاة erase(x): يحذف عنصراً محدداً إن وجده.
    • s.erase(5): تبحث عن الرقم 5 وتزيله نهائياً. وإذا لم يكن موجوداً من الأساس فلن يتسبب الأمر في أي أخطاء أو توقف للبرنامج.
    s.erase(5); // مسحنا الرقم 5 من الصندوق
  • زر التدمير الشامل clear(): يفرغ الصندوق تماماً بضربة واحدة.
    • s.clear(): تفريغ الحاوية بالكامل. دالة مهمة جداً في المسابقات عندما تحتاج لتصفير الحاوية قبل البدء في معالجة سيناريو جديد.
    s.clear(); // حذفنا كل العناصر بلا استثناء

المجموعة الثانية: أجهزة الرادار 📡

  • الرادار البسيط count(x): الأفضل للمبتدئين! يعيد 1 إذا وجد العنصر و 0 إذا لم يجده.
    • s.count(5): تسأل الحاوية كم مرة تكرر الرقم 5؟ وبما أن التكرار ممنوع، فستكون الإجابة إما 1 (أي أنه موجود) أو 0 (غير موجود).
    if (s.count(5)) {
    // الرقم موجود! نفذ الكود هنا
}
  • الرادار المتقدم find(x): يعيد "مؤشراً". إذا لم يجد العنصر سيصل للنهاية s.end().
    • s.find(5): دالة للمحترفين تبحث عن مكان الرقم.
    • != s.end(): تتأكد من أن عملية البحث لم تصل لخط النهاية الوهمي، مما يعني أننا اصطدنا العنصر المطلوب بنجاح!
    if (s.find(5) != s.end()) {
    // بحثنا ولم نصل للنهاية فارغي الأيدي، إذن هو موجود!
}

المجموعة الثالثة: شاشة المعلومات 🖥️

  • عداد العناصر size(): يخبرنا بعدد العناصر المختلفة.
    • s.size(): تعيد عدد العناصر الفعلي المخزن حالياً. لا تنسَ أن الأرقام المكررة لم يتم احتسابها.
    int total = s.size();
  • كاشف الفراغ empty(): للتحقق السريع هل الصندوق فارغ (true).
    • s.empty(): دالة سريعة تُعيد قيمة منطقية (True) إذا لم يكن هناك أي بيانات، وهي أسرع وأفضل أداءً من كتابة `s.size() == 0`.
    if (s.empty()) { /* الصندوق فارغ */ }

الدرس الرابع: المعركة التجريبية

محاكاة حية لدمج العمليات (الإنشاء، الإضافة، البحث، الحذف) في برنامج واحد.

سنبرمج نظاماً لتتبع اللاعبين النشطين في سيرفر لعبة. اللاعبون يدخلون (إضافة)، النظام يتأكد (بحث)، وبعضهم يغادر (حذف).

  • s.insert(...): تم استخدامها لإدخال اللاعبين للسيرفر. إدخال اللاعب رقم 3 مرة أخرى تم تجاهله بصمت.
  • s.count(3): استخدمنا الرادار السريع للتأكد من تواجد اللاعب رقم 3 قبل تنفيذ الأوامر.
  • s.erase(1): تم طرد أو حذف اللاعب رقم 1 من السيرفر.
  • for (int x : s): استخدمناها مرتين لطباعة القائمة ورؤية التغييرات الحية.
#include <iostream>
#include <set>
using namespace std;

int main() {
    set<int> s; // أنشأنا السيرفر
    
    // 1. اللاعبون يدخلون
    s.insert(3);
    s.insert(1);
    s.insert(4);
    s.insert(3); // لاعب يحاول الدخول مرتين بالخطأ (سيتم تجاهله)
    
    // 2. طباعة القائمة (ستظهر مرتبة)
    cout << "Elements: ";
    for (int x : s) cout << x << " ";
    cout << endl;
    
    // 3. البحث بالرادار
    if (s.count(3)) {
        cout << "3 is in the set" << endl;
    }
    
    // 4. خروج لاعب من السيرفر
    s.erase(1); // حذف الرقم 1
    
    cout << "After removing 1: ";
    for (int x : s) cout << x << " ";
    
    return 0;
}

شاشة المخرجات:

Elements: 1 3 4
3 is in the set
After removing 1: 3 4

💡 مسائل الاستعلامات الديناميكية

في المسابقات، الأسئلة التي تطلب منك تتبع أحداث متغيرة (يدخل، يخرج، هل هو موجود؟) تُسمى Dynamic Queries. هذا الكود البسيط هو الهيكل الأساسي لحل 90% من هذه المسائل بسرعة خيالية تمنع تجاوز الوقت (Time Limit)!

الدرس الخامس: رحلة الاستكشاف (Iterators)

تعلم كيفية المرور على جميع العناصر باستخدام طريقتين مختلفتين.

الطريقة الأولى: القطار السريع 🚆

الأسهل والأكثر راحة. يمر تلقائياً على كل محطة (عنصر) دون تعقيد:

  • تُسمى هذه الطريقة في لغة C++ بـ (Range-based for loop).
  • الأسلوب الأمثل والأسرع لطباعة كافة المحتويات دون الدخول في تفاصيل المؤشرات.
for (int x : s) {
    cout << x << " ";
}

نستخدمه لطباعة كل العناصر بسرعة وتجنب أخطاء الفهرسة.

الطريقة الثانية: المفتش المحترف (Iterator) 🕵️

في المسابقات المتقدمة نحتاج إلى "مفتش" يمسك بيده كشاف إضاءة (مؤشر) يتحرك خطوة بخطوة.

  • auto it: كلمة auto تجعل الكمبيوتر يستنتج نوع المفتش برمجياً لتوفير الوقت في المسابقات.
  • s.begin(): توجيه المؤشر للوقوف على أول عنصر في الصندوق.
  • s.end(): علامة نهاية الصندوق الوهمية (شرط التوقف).
  • it++: اطلب من المؤشر الانتقال للخطوة التالية.
  • *it: قراءة القيمة الفعلية الموجودة في المكان الذي يقف عليه المؤشر.
for (auto it = s.begin(); it != s.end(); it++) {
    cout << *it << " ";
}

الدرس السادس: عائلة الـ Sets

التعرف على الأنواع الثلاثة واختيار السلاح المناسب للمسألة.

1. الأخ الأكبر (set)

المنظم الصارم

  • ✅ مرتب دائماً
  • ❌ يرفض التكرار
  • ⚡ سرعة: O(log n)
  • النوع الأساسي الذي درسناه.
  • يستخدم شجرة متوازنة برمجياً لضمان ترتيب العناصر دائماً وحذف المكررات.
set<int> s;

2. الأخ الأوسط (multiset)

المنظم محب التجميع

  • ✅ مرتب دائماً
  • ✅ يسمح بالتكرار!
  • ⚡ سرعة: O(log n)
  • يحتفظ بجميع القيم حتى لو تكررت.
  • ممتاز جداً لترتيب درجات الطلاب حيث تتشابه بعض الدرجات ويجب الاحتفاظ بها جميعاً.
multiset<int> ms;

3. الأخ المتمرد (unordered_set)

البرق الفوضوي (Hashing)

  • ❌ غير مرتب (فوضوي)
  • ❌ يرفض التكرار
  • 🚀 سرعة: O(1) خارقة!
  • السلاح السري في المسابقات!
  • يستخدم تقنية الـ Hashing ليقوم بعمليات البحث والإضافة في وقت ثابت O(1) تقريباً.
  • نستخدمه إذا كان الترتيب غير مهم ونتعرض لخطر تجاوز الوقت (Time Limit).
unordered_set<int> us;

💡 القاعدة الذهبية الثلاثية للمسابقات

  1. ترتيب مطلوب + لا تكرار = استخدم set.
  2. ترتيب مطلوب + يوجد تكرار = استخدم multiset.
  3. الترتيب غير مهم + همك الوحيد السرعة القصوى = استخدم unordered_set.

الدرس السابع: آلة تنظيف البيانات

تحدي عملي: إزالة المكررات من المصفوفات وترتيبها بأقل عدد ممكن من الأسطر.

المشكلة: لدينا قائمة عادية (Vector) تحتوي على أرقام مكررة وعشوائية. مطلوب قائمة نهائية نظيفة ومرتبة. بدلاً من كتابة كود طويل ومعقد للبحث والحذف، سنصب القائمة في "المصفاة الذكية".

  • #include <vector>: أمر استدعاء القوائم العادية (التي تقبل العشوائية والتكرار).
  • nums: مصفوفة فوضوية بها الرقم 2 والرقم 3 مكرران.
  • unique.insert(x): بداخل الحلقة، نأخذ كل رقم ونرميه في حاوية set. الحاوية سترتب الأرقام فوراً وتطرد النسخ المكررة.
  • cout << x: في الحلقة الأخيرة، نقوم بعرض النتيجة النظيفة والمرتبة.
#include <iostream>
#include <set>
#include <vector> // أمر استدعاء القوائم العادية (تقبل الفوضى)
using namespace std;

int main() {
    // 1. القائمة الفوضوية
    vector<int> nums = {1, 2, 2, 3, 4, 3, 1};
    
    // 2. المصفاة الذكية (الصندوق)
    set<int> unique;
    
    // 3. عملية التنظيف (الصب داخل المصفاة)
    for (int x : nums) {
        unique.insert(x); // المصفاة سترفض التكرار وترتب تلقائياً
    }
    
    // 4. عرض النتيجة
    for (int x : unique) {
        cout << x << " ";
    }
    
    return 0;
}
